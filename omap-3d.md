# 3Dオリエンテーリングマップビューアー（MapLibre GL JS）開発要件定義書

## 1. プロジェクト概要
MapLibre GL JSを使用し、超高精度な3D地形データ（骨組み）の上に、オリエンテーリングのマップ（OMAP）を重ね合わせて表示する、完全無料のWeb GISアプリケーションのベース（HTML/JavaScript）を作成する。

## 2. 技術スタック
* **HTML5 / CSS3**: シンプルな1ファイル構成（またはJS/CSSを分ける標準構成）
* **JavaScriptライブラリ**: MapLibre GL JS
* **追加ライブラリ（将来用）**: PMTiles（将来的にCloudflare R2からPMTilesを読み込むための準備として、CDN経由で読み込んでおくこと）

## 3. 使用するデータソース（レイヤー構成）
地図は以下の順番で下から上に重ねること。

1.  **3D地形データ（Terrain RGB）**
    * 役割: 地形を立体的に立ち上げるための骨組み（標高データ）。
    * URL（例）: `https://cyberjapandata.gsi.go.jp/xyz/dem_png/{z}/{x}/{y}.png` （※地理院のTerrain-RGB規格。後からQ地図のURLに差し替えられるように変数化しておくこと）
2.  **背景地図（ベースマップ）**
    * 役割: 3D地形の上に貼り付ける皮（テクスチャ）。
    * URL: Q地図のCS立体図等のXYZタイル URL（プレースホルダーとして `https://cyberjapandata.gsi.go.jp/xyz/pale/{z}/{x}/{y}.png` などを仮置きし、後で変更しやすくすること）
3.  **オーバーレイ地図（OMAP）**
    * 役割: 最上位に半透明で表示するオリエンテーリングマップ。
    * URL: プレースホルダーとして任意のXYZタイルURLを仮置きし、不透明度（opacity）を `0.7` 程度に設定すること。

## 4. 実装すべき機能要件
* **フルスクリーン表示**: 地図がブラウザの画面いっぱいに表示されること。
* **3Dコントロールの追加**: ユーザーがマウスの右クリック（またはCtrl+ドラッグ）で画面の角度（Pitch）や方角（Bearing）を自由に変更できること。
* **初期視点の設定**: 
    * Pitch（傾き）を `60` 度程度に設定し、開いた瞬間に「3Dの立体感」がわかるようにすること。
    * Zoomレベルは `14` 程度。初期座標は日本の任意の山間部（例：`[136.226, 34.689]` 等）に設定すること。
* **地形の誇張（Exaggeration）**: 地形の起伏を分かりやすくするため、標高の強調係数を `1.2` 〜 `1.5` 程度に設定すること。

# 3Dオリエンテーリングマップビューアー（ローカルKMZ読み込み機能付き）開発要件定義書

## 1. プロジェクト概要
MapLibre GL JSを使用し、超高精度な3D地形データ（骨組み）の上に、ユーザーが手元の端末からアップロード（またはドラッグ＆ドロップ）した「KMZファイル」をブラウザ上で解凍・解析し、地図上にピッタリと重ね合わせるWeb GISアプリケーションを作成する。サーバーサイドの処理は持たず、完全なクライアントサイド（ブラウザ内）で完結させること。

## 2. 技術スタックとライブラリ
* **ベース**: 素のHTML5 / CSS3 / Vanilla JavaScript（React等は使用不可）
* **地図描画**: MapLibre GL JS (CDNから読み込む)
* **KMZ（ZIP）解凍処理**: JSZip (CDNから読み込む。例: `https://cdnjs.cloudflare.com/ajax/libs/jszip/3.10.1/jszip.min.js`)

## 3. マップの初期設定（ベースマップと3D地形）
* **初期視点**: 日本の山間部（例: 経度 `136.226`, 緯度 `34.689`）、ズーム `14`、Pitch `60`度。
* **背景地図レイヤー（下敷き）**:
  * 地理院地図の淡色地図（`https://cyberjapandata.gsi.go.jp/xyz/pale/{z}/{x}/{y}.png`）を `raster` レイヤーとして追加。
* **3D地形（Terrain-RGB）**:
  * 地理院の標高タイル（`https://cyberjapandata.gsi.go.jp/xyz/dem_png/{z}/{x}/{y}.png`）を使用し、`terrain` を有効化する。標高の強調係数（exaggeration）は `1.2` とする。

## 4. UI/UX要件
* 画面全体をフルスクリーンで地図表示（`width: 100vw`, `height: 100vh`）。
* 画面の左上（または右下）に、目立つように「KMZファイルをアップロード」できるUIを配置する。
  * ファイル選択ボタン（`<input type="file" accept=".kmz">`）
  * 画面全体へのドラッグ＆ドロップにも対応させること。

## 5. KMZファイルの読み込みと描画ロジック（最重要）
KMZがアップロードされたら、以下の手順でJavaScriptで処理を行うこと。

1. **JSZipによる解凍**:
   * アップロードされたファイルをJSZipで読み込み、中のファイル一覧を取得する。
2. **KMLと画像ファイルの特定**:
   * 拡張子が `.kml` のファイルと、画像ファイル（`.jpg`, `.png` 等）を見つける。
3. **KMLの解析（DOMParser）**:
   * KMLのテキストデータをXMLとしてパースする。
   * `<GroundOverlay>` タグを探し、その中の `<LatLonBox>` から `<north>`, `<south>`, `<east>`, `<west>` の値（緯度経度）を取得する。
4. **画像データのURL変換**:
   * 抽出した画像ファイルをBlobに変換し、`URL.createObjectURL()` を使ってブラウザ内で表示可能な一時URLを生成する。
5. **MapLibreへのImage Source追加**:
   * MapLibreの `addSource` を使い、`type: 'image'` として追加する。
   * `url` には先ほどの ObjectURL を指定する。
   * `coordinates` には、取得した `<LatLonBox>` の値から、**[ [西, 北], [東, 北], [東, 南], [西, 南] ]** （Top-Left, Top-Right, Bottom-Right, Bottom-Left）の順番で配列を作成して渡すこと。
6. **Raster Layerの追加**:
   * 追加したソースを元に、`type: 'raster'` レイヤーを追加する。不透明度（`raster-opacity`）は `0.7` に設定する。
   * 地図の視点（カメラ）を、追加した画像の座標（Bounding Box）へ `fitBounds` などを使って自動で移動させること。
